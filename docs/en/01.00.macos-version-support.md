# macOS Version Support Plan

## Goal

Update Agent Battery's build configuration so the app can run on mainstream macOS versions while keeping the implementation conservative. The target minimum runtime version is **macOS 14.0+**.

This avoids extra compatibility fallbacks for current app APIs such as SwiftUI settings presentation, while still removing the accidental `macOS 26.3` deployment lock.

## Scope

In scope:

- Lower the app and test deployment target to `macOS 14.0`.
- Keep the app as a macOS-only menu bar app.
- Update English and Chinese README build requirements.
- Validate Debug, test, and Release universal build paths.

Out of scope:

- Support for macOS 13 or earlier.
- Code signing, notarization, or release distribution policy changes.
- UI redesign or data-source behavior changes.

## Current state

The Xcode project currently sets `MACOSX_DEPLOYMENT_TARGET = 26.3` in project-level and test target build settings. README files also say source builds require macOS 15+ and Xcode 26.

The app uses APIs that fit a `macOS 14.0+` baseline:

- `MenuBarExtra` is available from macOS 13.
- `SMAppService.mainApp` is available from macOS 13.
- SwiftUI `openSettings` is available from macOS 14.

## Action items

[ ] Update `agent-battery.xcodeproj/project.pbxproj` so all explicit `MACOSX_DEPLOYMENT_TARGET` values are `14.0`.

[ ] Check app target settings in both Debug and Release configurations, including generated Info.plist output and `LSMinimumSystemVersion`.

[ ] Check unit test and UI test target settings so tests build against the same minimum macOS version.

[ ] Keep `script/build_and_run.sh` and `script/release_build.sh` unchanged unless validation shows a deployment-target-specific issue.

[ ] Update `README.md` and `README.zh.md` to document `macOS 14.0+` as the runtime baseline and current Xcode as the recommended build toolchain.

[ ] Build Debug with `xcodebuild -project agent-battery.xcodeproj -scheme agent-battery -configuration Debug build`.

[ ] Run tests with `xcodebuild test -project agent-battery.xcodeproj -scheme agent-battery -destination 'platform=macOS'`.

[ ] Build Release with `script/release_build.sh build` and confirm the universal binary still contains `arm64` and `x86_64`.

[ ] Inspect the built app metadata to confirm the minimum system version is `14.0`.

## Validation checklist

- Debug build succeeds without availability errors.
- Unit tests and UI tests compile under the new deployment target.
- Release universal build still passes `lipo` verification.
- README requirements match the actual deployment target.
- No code paths claim support for macOS 13 or earlier.

## Risks

- Newer Xcode project metadata may still require a recent Xcode to open or build the project, even though the app runtime supports macOS 14.0+.
- If future UI work adds macOS 15+ or macOS 26+ APIs, those calls must be guarded or the deployment target must be revisited.
