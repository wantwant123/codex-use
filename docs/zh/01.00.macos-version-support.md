# macOS 版本支持计划

## 目标

调整 Agent Battery 的编译配置,让应用支持主流 macOS 版本,同时保持实现策略保守。本次最低运行版本设为 **macOS 14.0+**。

这个目标可以去掉当前意外锁定到 `macOS 26.3` 的限制,也避免为了支持更旧系统而引入额外兼容分支。

## 范围

范围内:

- 将 App 和测试 target 的 deployment target 下调到 `macOS 14.0`。
- 保持应用仍是 macOS-only 菜单栏应用。
- 更新英文与中文 README 中的构建/运行要求。
- 验证 Debug、测试和 Release universal 构建路径。

范围外:

- 支持 macOS 13 或更早版本。
- 签名、公证或发布分发策略调整。
- UI 重设计或数据源行为变更。

## 当前状态

Xcode 项目当前在项目级和测试 target 配置中显式设置了 `MACOSX_DEPLOYMENT_TARGET = 26.3`。README 也写着源码构建需要 macOS 15+ 与 Xcode 26。

现有应用 API 适合以 `macOS 14.0+` 作为保守基线:

- `MenuBarExtra` 从 macOS 13 开始可用。
- `SMAppService.mainApp` 从 macOS 13 开始可用。
- SwiftUI `openSettings` 从 macOS 14 开始可用。

## 执行项

[ ] 更新 `agent-battery.xcodeproj/project.pbxproj`,将所有显式 `MACOSX_DEPLOYMENT_TARGET` 改为 `14.0`。

[ ] 检查 App target 的 Debug 和 Release 配置,确认生成的 Info.plist 与 `LSMinimumSystemVersion` 符合预期。

[ ] 检查单元测试和 UI 测试 target 配置,确保测试也使用同一个最低 macOS 版本。

[ ] 暂不修改 `script/build_and_run.sh` 和 `script/release_build.sh`,除非验证时发现与 deployment target 相关的问题。

[ ] 更新 `README.md` 与 `README.zh.md`,说明运行基线为 `macOS 14.0+`,构建工具链使用当前 Xcode 或推荐的新版 Xcode。

[ ] 使用 `xcodebuild -project agent-battery.xcodeproj -scheme agent-battery -configuration Debug build` 验证 Debug 构建。

[ ] 使用 `xcodebuild test -project agent-battery.xcodeproj -scheme agent-battery -destination 'platform=macOS'` 验证测试构建与运行。

[ ] 使用 `script/release_build.sh build` 验证 Release universal 构建,并确认产物仍包含 `arm64` 和 `x86_64`。

[ ] 检查构建产物元数据,确认最低系统版本为 `14.0`。

## 验证清单

- Debug 构建成功,没有 availability 错误。
- 单元测试和 UI 测试能在新 deployment target 下编译。
- Release universal 构建仍通过 `lipo` 验证。
- README 中的要求与实际 deployment target 一致。
- 不在任何代码路径中声称支持 macOS 13 或更早版本。

## 风险

- 即使应用运行时支持 macOS 14.0+,较新的 Xcode project metadata 仍可能要求较新的 Xcode 才能打开或构建项目。
- 如果后续 UI 开发新增 macOS 15+ 或 macOS 26+ API,需要为这些调用增加可用性保护,或重新评估 deployment target。
